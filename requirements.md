# Требования к Learn Helper

## 1. Назначение

Learn Helper предназначен для изучающего чтения учебных материалов. Сервис должен превращать загруженный PDF, TXT или Markdown в удобную mdBook-книгу, помогать пользователю читать материал, получать краткие конспекты, проходить самопроверку, задавать вопросы AI-помощнику и при необходимости слушать текст.

Главная цель - не просто показать документ, а поддержать цикл обучения: чтение, понимание, повторение, проверка, возвращение к сложным местам.

## 2. Основной сценарий

1. Пользователь авторизуется, если задан `AUTH_PASSWORD`.
2. Пользователь загружает материал или открывает ранее загруженный источник из библиотеки.
3. Backend сохраняет исходник и собирает mdBook.
4. Frontend открывает собранную книгу в iframe.
5. При открытии раздела в конце текста появляется краткий конспект.
6. Пользователь может открыть помощника и сформировать тренажер по текущему разделу, нескольким разделам или целой главе.
7. Пользователь проходит тесты, практическое задание и открытый вопрос.
8. AI оценивает ответы и дает обратную связь.
9. Пользователь может включить озвучку раздела, следить за подсветкой текущего чтения, менять скорость или скачать аудио.

## 3. Поддерживаемые форматы

Сервис должен принимать:

- `.pdf`;
- `.txt`;
- `.md`;
- `.markdown`.

PDF должен обрабатываться через доступные CLI:

- `pdf2md`, если установлен;
- fallback: `pdftotext -layout`;
- дополнительная нормализация: `pandoc`, если установлен.

Если PDF содержит только сканированные страницы без текстового слоя, сервис может вернуть ошибку. OCR является будущим расширением.

## 4. Сборка mdBook

Сервис должен:

- сохранять исходный файл в `backend/data/sources`;
- конвертировать источник в Markdown;
- создавать структуру mdBook в `backend/data/books/<bookId>`;
- разбивать Markdown на главы и разделы по заголовкам;
- генерировать `SUMMARY.md`;
- подключать дополнительные CSS/JS helper-файлы;
- запускать `mdbook build`;
- открывать готовую книгу через `/api/books/view/:bookId`.

### Требования к качеству PDF-разбиения

- Раздел не должен обрываться из-за колонтитула на следующей странице.
- Верхние повторяющиеся заголовки страницы не должны восприниматься как начало нового раздела.
- Если на новой странице сначала идет running header, а затем продолжение предыдущего абзаца, этот текст должен остаться в предыдущем разделе.
- На примере `sql_primer.pdf` раздел `1.1. Что такое базы данных и зачем они нужны` не должен заканчиваться фразой `Они могут получить доступ к базе`; продолжение `данных, используя прикладные программы...` должно оставаться в разделе 1.1.

## 5. Конспекты

При открытии раздела сервис должен показывать конспект в конце текста, чтобы он не мешал чтению.

Конспект должен содержать:

- краткое резюме;
- ключевые тезисы;
- важные термины с определениями, если они есть.

Требования:

- конспект загружается автоматически при открытии раздела;
- уже готовый конспект не должен генерироваться повторно;
- frontend может хранить результат в `localStorage`;
- backend должен хранить результат на диске рядом с книгой;
- ключ кэша должен учитывать `bookId`, путь раздела и hash текста;
- если текст раздела изменился после пересборки, конспект должен быть создан заново.

## 6. AI-помощник и тренажер

Помощник должен работать поверх структуры mdBook.

Пользователь может сформировать вопросы:

- по одному разделу;
- по нескольким выбранным разделам;
- по всей главе.

AI должен вернуть структурированный JSON:

- 3 тестовых вопроса с вариантами ответов;
- 1 практическое задание;
- 1 открытый вопрос;
- подсказки к каждому заданию.

Тренажер должен:

- показывать задания в отдельной вкладке;
- сохранять готовые наборы вопросов в браузере для текущей книги;
- не генерировать заново уже созданный набор с тем же источником;
- проверять практические и открытые ответы через AI;
- давать короткую доброжелательную обратную связь.

## 7. Чат

Сервис должен предоставлять простой AI-чат для вопросов пользователя по материалу. Чат может использовать переданный контекст, но не должен блокировать чтение книги.

## 8. TTS-озвучка

Сервис должен подключать TTS в toolbar mdBook рядом с левыми кнопками.

Меню озвучки должно включать:

- сделать аудио / пауза / продолжить;
- остановить аудио;
- скролить за читающим;
- выделять текущее чтение;
- выбор скорости;
- скачать аудио.

Требования к обработке больших текстов:

- большой раздел не должен отправляться в TTS одним запросом;
- frontend должен дробить текст на небольшие фрагменты;
- аудио должно начинать воспроизводиться по мере готовности первых фрагментов;
- уже готовые фрагменты должны кэшироваться в IndexedDB;
- при переходе в другой раздел генерация и воспроизведение должны прерываться;
- скачивание должно догенерировать недостающие фрагменты и собрать один WAV-файл.

Требования к подсветке:

- при включенной подсветке должен выделяться текущий читаемый блок;
- дополнительно должны подсвечиваться текущие 2-3 слова на основе `timecodes`, возвращаемых TTS;
- при остановке, паузе или переходе подсветка должна очищаться.

## 9. Хранение данных

Сервис должен хранить:

- исходные файлы;
- собранные книги;
- структуру mdBook;
- кэш конспектов;
- прогресс пользователя;
- локальные наборы вопросов в браузере;
- TTS-аудиофрагменты в IndexedDB.

Нельзя терять пользовательские данные при перезапуске backend.

## 10. Авторизация

Если задан `AUTH_PASSWORD`, backend должен защищать API.

Допускаются:

- cookie-based авторизация после `/api/auth`;
- `Authorization: Bearer <password>` для запросов frontend.

Если `AUTH_PASSWORD` не задан, сервис может работать без авторизации в локальном режиме.

## 11. Нефункциональные требования

- Работа с большими файлами не должна блокировать интерфейс без явного индикатора.
- AI-запросы должны быть ограничены по объему текста.
- Повторные AI-запросы должны кэшироваться там, где это возможно.
- Ошибки генерации книги, AI и TTS должны показываться пользователю понятным текстом.
- Backend должен уметь запускаться из systemd, где `PATH` может отличаться от интерактивной shell.
- Для `mdbook` должен поддерживаться явный путь через `MDBOOK_BIN`.
- Сборка backend TypeScript должна проходить без ошибок.
- Frontend lint должен проходить без ошибок.

## 12. Текущая реализация

Реализовано:

- Next.js frontend;
- Express + TypeScript backend;
- FastAPI TTS сервис;
- библиотека источников;
- загрузка PDF/TXT/Markdown;
- сборка mdBook;
- исправление ложных PDF running headers при разбиении разделов;
- открытие книги в iframe;
- helper-скрипт mdBook для конспекта и TTS;
- серверный кэш конспектов;
- вопросы и тренажер по разделам/главам;
- AI-проверка ответов;
- чат;
- TTS по чанкам, пауза, остановка, скорость, скачивание, подсветка слов.

## 13. Будущие улучшения

- OCR fallback для сканированных PDF;
- более точное восстановление таблиц из PDF;
- серверный кэш наборов вопросов;
- синхронизация прогресса чтения между браузерами;
- настройка голоса TTS;
- полноценный streaming endpoint для TTS вместо последовательных небольших запросов.
