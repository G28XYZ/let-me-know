# Требования к сервису для изучающего чтения с AI-помощником

## 1. Назначение сервиса

Сервис предназначен для изучающего чтения текстовых материалов. Пользователь загружает текстовый файл или PDF, читает материал внутри сервиса, выбирает метод изучения, а AI-помощник анализирует текущие фрагменты текста, помогает выделять главное, формирует конспект, предлагает задания, вопросы и точки повторения.

Главная задача сервиса - помогать пользователю не просто читать материал, а понимать, перерабатывать и закреплять знания.

## 2. Основной пользовательский сценарий

1. Пользователь загружает материал в формате `.txt` или `.pdf`.
2. Сервис отображает материал в режиме ридера.
3. Перед началом чтения пользователь выбирает метод изучения:
   - SQ3R;
   - конспектирование;
   - метод Фейнмана;
   - другие методы в будущем.
4. Пользователь начинает читать материал.
5. Сервис определяет текущий видимый фрагмент текста.
6. Текущий фрагмент отправляется AI-помощнику через API.
7. AI анализирует фрагмент и возвращает:
   - на что обратить внимание;
   - краткий смысловой разбор;
   - важные термины и идеи;
   - элементы конспекта;
   - вопросы и задания в рамках выбранного метода.
8. По мере чтения AI определяет моменты, где стоит сделать паузу для повторения.
9. Для продолжения чтения пользователь должен выполнить тест, задание или повторение по предыдущему изученному отрывку.
10. Сервис сохраняет прогресс, ответы, конспекты, повторения и проблемные места.

## 3. Поддерживаемые форматы материалов

Сервис должен поддерживать загрузку:

- `.txt`;
- `.pdf`.

Для PDF требуется извлечение текста и отображение материала в удобном для чтения виде. Если PDF не содержит текстового слоя, в будущем может потребоваться OCR.

## 4. Режим чтения

Сервис должен отображать материал в формате книги или ридера.

Основные требования:

- удобное чтение текста;
- отображение текущей позиции;
- сохранение прогресса чтения;
- возможность вернуться к прошлым фрагментам;
- отображение AI-подсказок рядом с материалом;
- доступ к конспекту по материалу;
- возможность сменить выбранный метод изучения во время чтения.

При смене метода сервис должен продолжить работу с текущей позиции. Новые задания, подсказки и элементы конспекта должны строиться уже по новому методу.

## 5. AI-помощник

AI-помощник подключается через API. Вариант с iframe ChatGPT можно рассматривать как экспериментальный, но основным вариантом следует считать API, потому что сервису нужно контролировать контекст, сохранять ответы, строить задания и фиксировать прогресс.

AI должен получать:

- текущий видимый фрагмент текста;
- предыдущий фрагмент с момента последней паузы;
- выбранный метод изучения;
- историю краткого прогресса пользователя;
- уже сформированные заметки и ответы пользователя.

AI должен возвращать структурированный ответ:

- краткое резюме фрагмента;
- ключевые идеи;
- важные термины;
- предупреждения о сложных местах;
- вопросы для самопроверки;
- задания по выбранному методу;
- рекомендацию: продолжать чтение или сделать паузу;
- материал для конспекта.

## 6. Паузы и разблокировка чтения

Сервис должен уметь временно останавливать чтение в точках повторения.

Отрывок для повторения определяется как текст между предыдущей паузой и текущей паузой.

Чтобы продолжить чтение, пользователь проходит через цепочку заданий (формируется ИИ в зависимости от контента):

- **Тесты (Quiz):** выбор правильных вариантов ответа из предложенных.
- **Практическое задание:** выполнение конкретной задачи по прочитанному.
- **Закрепление:** ответ на открытый вопрос или пересказ своими словами.

После выполнения задания AI-преподаватель оценивает результат:

- разрешает продолжить (если ответ верный);
- просит дополнить/исправить ответ, выделяет ошибки и предлагает кнопку для получения подсказки с правильным решением.

## 7. Метод SQ3R

Сервис должен поддерживать этапы SQ3R:

### 7.1. Survey

Перед чтением AI делает обзор материала:

- структура;
- предполагаемые темы;
- ключевые разделы;
- сложные места.

### 7.2. Question

AI помогает сформулировать вопросы к материалу:

- что нужно понять;
- какие тезисы проверить;
- какие понятия встретятся.

### 7.3. Read

Во время чтения AI подсказывает:

- главные идеи;
- связи между частями;
- что важно не пропустить.

### 7.4. Recite

В паузах сервис просит пользователя пересказать прочитанное своими словами.

### 7.5. Review

После раздела или сессии сервис предлагает повторение:

- тесты;
- карточки;
- краткий обзор;
- вопросы по слабым местам.

## 8. Метод конспектирования

Сервис должен помогать превращать текст в знания, а не просто копировать его.

Поддерживаемые форматы:

- линейный конспект;
- интеллект-карта;
- карточки;
- таблицы;
- Zettelkasten-заметки.

Главное правило метода: пользователь должен перерабатывать мысль своими словами.

AI должен:

- выделять идеи для конспекта;
- предлагать формулировки;
- просить пользователя переформулировать;
- связывать новые заметки с предыдущими;
- создавать карточки для повторения;
- отмечать места, где пользователь просто копирует текст.

## 9. Метод Фейнмана

Сервис должен помогать пользователю проверять настоящее понимание материала.

Основной цикл:

1. Пользователь читает фрагмент.
2. Сервис просит объяснить прочитанное простыми словами.
3. AI анализирует объяснение.
4. AI находит пробелы, неточности и сложные места.
5. Сервис предлагает перечитать нужный фрагмент или дает уточняющие вопросы.
6. Пользователь повторяет объяснение.

Критерий метода: если пользователь не может объяснить материал простыми словами, материал считается недостаточно понятым.

## 10. Данные для хранения

Сервис должен сохранять:

- загруженные материалы;
- извлеченный текст;
- позицию чтения;
- выбранный метод;
- историю смены методов;
- AI-конспект;
- пользовательские заметки;
- вопросы и ответы;
- результаты тестов;
- точки пауз;
- проблемные темы;
- статистику прогресса.

## 11. Текущая версия

В текущей версии реализовано:

- **Архитектура:** Клиент-серверное приложение (Frontend: Next.js + Tailwind CSS, Backend: Express + TypeScript).
- Загрузка `.txt` и `.pdf` с автоматической склейкой переносов слов.
- Ридер для чтения с выводом примерного времени на фрагмент.
- Выбор метода изучения перед началом (SQ3R, конспектирование, метод Фейнмана).
- Разбивка текста на смысловые фрагменты с возможностью пропуска служебных разделов (оглавления, введения).
- Передача текущего фрагмента в AI через API (с поддержкой локальных OpenAI-совместимых серверов и Gemini CLI).
- AI-панель с подсказками, конспектом и сохранением истории.
- Многоступенчатые паузы после блоков: Тесты -> Практика -> Свободный ответ.
- Автоматическая проверка ответов пользователя с помощью ИИ-преподавателя (оценка правильности, блокировка кнопки далее, выдача подсказок).
- Защита от переключения фрагментов во время генерации ответов ИИ.

## 12. Нефункциональные требования

- Сервис должен работать быстро даже с большими файлами.
- AI-запросы должны отправляться не на каждый скролл, а по устойчивому изменению видимого фрагмента или по достижению нового блока.
- Нужно ограничивать объем текста, отправляемого в AI, чтобы не превышать лимиты модели.
- Ответы AI должны сохраняться в структурированном виде.
- Пользователь должен понимать, какой фрагмент текста сейчас анализируется.
- Должна быть возможность отключить автоматические паузы или настроить их частоту.
