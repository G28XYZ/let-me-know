# Learn Helper

Сервис для изучающего чтения с AI-помощником.

## Запуск без AI API

```bash
python3 -m http.server 4173
```

После запуска открыть:

```text
http://127.0.0.1:4173
```

В этом режиме поле `Backend endpoint` нужно оставить пустым. Будет использоваться локальный демо-анализ без внешней нейронки.

## Запуск с OpenAI API

Ключ API должен храниться на сервере, а не во фронтенде.

```bash
OPENAI_API_KEY=sk-... node server.mjs
```

После запуска открыть:

```text
http://127.0.0.1:4173
```

В настройках приложения:

1. Указать endpoint:

```text
/api/analyze
```

2. Нажать `Проверить ИИ`.
3. После успешной проверки загрузить материал.

Можно выбрать модель через переменную окружения:

```bash
OPENAI_API_KEY=sk-... OPENAI_MODEL=gpt-5-mini node server.mjs
```

## Запуск с локальной OpenAI-compatible моделью

Backend `/api/analyze` умеет ходить в OpenAI-compatible endpoint:

```text
POST {OPENAI_BASE_URL}/chat/completions
```

Важно: в интерфейсе приложения нельзя указывать `http://127.0.0.1:11434/v1/chat/completions` напрямую. В поле `Backend endpoint` всегда указывается только:

```text
/api/analyze
```

`OPENAI_BASE_URL` задается при запуске `server.mjs`.

### Ollama

Пример:

```bash
ollama serve
ollama pull qwen3:14b
OPENAI_BASE_URL=http://127.0.0.1:11434/v1 OPENAI_MODEL=qwen3:14b node server.mjs
```

Если Qwen возвращает текст вместо JSON, backend автоматически добавляет `/no_think`, просит JSON mode и использует fallback-разбор, чтобы интерфейс не падал с ошибкой.

### LM Studio

1. Запустить локальный server в LM Studio.
2. Загрузить модель, например Qwen3.
3. Запустить приложение:

```bash
OPENAI_BASE_URL=http://127.0.0.1:1234/v1 OPENAI_MODEL=qwen3-8b node server.mjs
```

### llama.cpp server

Пример формата запуска приложения:

```bash
OPENAI_BASE_URL=http://127.0.0.1:8080/v1 OPENAI_MODEL=qwen3 node server.mjs
```

В приложении в поле `Backend endpoint` указать:

```text
/api/analyze
```

Затем нажать `Проверить ИИ`.

## Как работает AI-подготовка

После загрузки файла сервис сначала отправляет в backend `/api/prepare` только первые 30 страниц. Это нужно, чтобы пользователь не ждал обработку всей книги.

ИИ должен:

- разбить материал на учебные фрагменты;
- распознать оглавление;
- распознать введение;
- кратко проанализировать оглавление и введение;
- пометить служебные части как пропускаемые;
- подготовить основной материал для чтения по выбранному методу.

Когда пользователь подходит к концу уже подготовленных фрагментов, frontend догружает следующие 30 страниц через `/api/prepare`.

Пока идет подготовка или анализ фрагмента, интерфейс блокируется loader'ом, чтобы пользователь не сменил метод, файл или позицию чтения в середине AI-запроса.

В ридере отображается один активный фрагмент целиком. Если фрагмент большой, внутри ридера появляется скролл. Назад и вперед можно переключаться между уже подготовленными фрагментами, чтобы вернуться к началу.

Прогресс чтения считается по страницам: текущая пройденная страница относительно общего числа страниц документа. Для PDF используются реальные страницы, для TXT страницы считаются приблизительно по объему текста.

## Запуск с Gemini CLI

Если установлен `gemini` CLI, можно выбрать provider `Gemini CLI` в настройках приложения.

Проверка:

```bash
gemini --version
```

Запуск приложения:

```bash
node server.mjs
```

В интерфейсе:

1. Открыть `Настройки API`.
2. В `AI provider` выбрать `Gemini CLI`.
3. Нажать `Проверить ИИ`.
4. При необходимости открыть панель `Gemini CLI` и выполнить `help`, `version`, `sessions` или тестовый prompt.

Backend использует безопасный режим запуска CLI: разрешены только команды Gemini CLI, а не произвольный shell. Для анализа материала используется headless-режим:

```bash
gemini --prompt "<prompt>" --output-format text --skip-trust
```

Диагностика Gemini CLI доступна в интерфейсе: `Настройки API` -> `Диагностика Gemini CLI`.
Она показывает:

- используемый `GEMINI_BIN`;
- модель из `GEMINI_MODEL`, которая реально отправляется в `gemini --model`, или пометку `CLI default`, если backend модель не задает;
- лимиты известной модели Gemini Pro/Flash/Lite: input/output token limit;
- команду, которой backend отправляет текст в CLI;
- лимиты приложения для разных задач (`prepare`, `analyze`, `questions`, `evaluate`);
- последние запуски CLI с кратким preview prompt;
- быстрый статус без долгих обращений к модели.

То же самое можно получить напрямую:

```bash
curl http://127.0.0.1:4173/api/gemini/diagnostics
```

Важно: `gemini --help` не отдает фактическую default-модель CLI. Чтобы helper показывал точную используемую модель и ее лимиты, задайте `GEMINI_MODEL`, например `gemini-2.5-pro`, `gemini-2.5-flash` или `gemini-2.5-flash-lite`.

Можно задать путь и модель:

```bash
GEMINI_BIN=gemini GEMINI_MODEL=gemini-2.5-pro node server.mjs
```

Если CLI требует аутентификацию, сначала выполните настройку в обычном терминале командой `gemini`, затем вернитесь в веб-интерфейс и нажмите `Проверить ИИ`.

## Почему нельзя указать chatgpt.com

`https://chatgpt.com/` - это веб-интерфейс, а не API endpoint. Браузер блокирует такие запросы по CORS, а также нельзя безопасно передавать API key из клиентского кода.

Правильная схема:

```text
браузер -> /api/analyze на вашем backend -> OpenAI-compatible API
```

Вариант с iframe для `chatgpt.com` тоже не подходит: сайт запрещает встраивание, а браузер не даст читать или заполнять поле ввода внутри cross-origin iframe через `querySelector`.

## Что уже есть

- загрузка `.txt` и `.pdf`;
- ридер с разбиением материала на фрагменты;
- выбор метода изучения: SQ3R, конспектирование, метод Фейнмана;
- смена метода во время чтения;
- AI-подготовка документа через `/api/prepare`;
- AI-анализ текущего фрагмента через `/api/analyze`;
- AI-панель с резюме, фокусом внимания, вопросом и рекомендацией;
- автоматический конспект;
- паузы повторения;
- задания для разблокировки дальнейшего чтения;
- опциональный backend endpoint `/api/analyze`;
- поддержка OpenAI-compatible local endpoint через `OPENAI_BASE_URL`.

## PDF

PDF извлекается через PDF.js с CDN. Если библиотека недоступна или PDF не содержит текстового слоя, файл отображается как референс, но текст может не извлечься.
